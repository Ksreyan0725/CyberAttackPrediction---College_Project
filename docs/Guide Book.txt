CYBER ATTACK PREDICTION - COMPREHENSIVE PROJECT GUIDE (UPDATED 2026)
=====================================================================

Welcome to the Cyber Attack Prediction project! This guide covers the modernized architecture, datasets, ML/DL models, Generative AI integration, and instructions for this system.

1. PROJECT OVERVIEW & ARCHITECTURE
----------------------------------
This project is an advanced Machine Learning and Deep Learning-based Intrusion Detection System (IDS) that classifies network traffic as "Normal" or a specific "Cyber Attack".

The project now features a hybrid architecture:
   A. Research Environment (Jupyter Notebooks): For data analysis, model comparison, and Deep Learning experiments.
   B. Production Web Application (Flask + Bootstrap 5): A modernized, responsive UI for real-time predictions.
   C. Model Persistence Layer: A dedicated training script (train_model.py) to save models, ensuring the web app is fast and efficient.
   D. Generative AI Insight Engine: Automatically generates natural language explanations and mitigation steps for detected attacks.


2. EXTENSIVE DATASET ANALYSIS
-----------------------------
The system evaluates models across four major cybersecurity datasets:
   * NSL-KDD (kdd_train.csv): Primary dataset for the web app.
   * CIC-IDS-2017: Modern benign and common attack traffic.
   * CIC-DDoS-2019: Focused on DDoS attack patterns.
   * X-IIoTID: Specific to Industrial Internet of Things (IIoT) intrusion.

Preprocessing Steps:
   - Label Encoding: Converts categorical data to numeric.
   - Imputation: Handles missing values with means or zeros.
   - Normalization: Scales features using StandardScaler.


3. AI & MACHINE LEARNING MODELS
-------------------------------
* Traditional ML & Deep Learning:
  - Random Forest, KNN, SVM, MLP, Logistic Regression, Naive Bayes.
  - Deep Neural Networks (DNN) built with Keras/TensorFlow.

* Generative AI Integration (NEW):
  - The project now includes a GenAI Insight Engine. After a prediction is made, the system provides a detailed AI-generated explanation of the threat and recommended security measures.

* Explainable AI (XAI):
  - Uses SHAP (SHapley Additive exPlanations) in the notebooks to visualize which network features impacted the model's decision.


4. FOLDER STRUCTURE
-------------------
In the CyberAttackPrediction folder:
   * Main.py: The Flask backend server (optimized for model loading).
   * train_model.py (NEW): Run this to train and save the ML model to disk.
   * .env (NEW): Secure storage for administrator credentials and secret keys.
   * templates/base.html (NEW): Modern Jinja2 base template with Bootstrap 5.
   * model/trained_rf_model.pkl (NEW): The persistent, pre-trained model file.
   * requirements.txt: Updated with all dependencies including python-dotenv.

In the main Project folder:
   * .venv: Isolated Python 3.7.2 environment containing all libraries.
   * .gitignore: Configured to keep bulky datasets and sensitive .env files safe.


5. HOW TO RUN THE PROJECT
-------------------------

STEP 1: INITIAL SETUP (First Time Only)
   1. Open a terminal in the project root.
   2. Install dependencies: `.venv\Scripts\python.exe -m pip install -r CyberAttackPrediction\requirements.txt`
   3. Train the model: `.venv\Scripts\python.exe CyberAttackPrediction\train_model.py`

STEP 2: RUNNING THE PROJECT

A. Research & Notebooks:
   1. Double-click "Start_Jupyter_Venv.bat" in the main folder.
   2. Open "ExtensionCyberAttack.ipynb" or "ProposeCyberAttack.ipynb".
   3. Run cells to view graphs, model comparisons, and SHAP plots.

B. User-Facing Web Application:
   1. Double-click "Start_WebApp_Venv.bat" in the main folder.
   2. Wait for "Running on http://127.0.0.1:5000/".
   3. Open http://127.0.0.1:5000 in your browser.
   4. Login using the credentials stored in your .env file (default: admin/admin).
   5. Upload "testData.csv" to see real-time predictions and GenAI insights.


IMPORTANT: Always use the .bat files or the .venv path to ensure you are using the correct Python version and libraries.
