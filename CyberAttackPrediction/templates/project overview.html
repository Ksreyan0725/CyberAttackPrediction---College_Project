Project Overview

This project, "Cyber Attack Prediction: From Traditional ML to Generative AI," is designed to protect digital infrastructure by identifying malicious network traffic patterns before they cause damage.

The Problem

Traditional security systems rely on "signatures" (lists of known bad actors). They fail when a new, unknown attack occurs.

The Solution

Our system uses AI to learn behavior. It can spot a hacker even if it has never seen that specific attack before.

The School Gate Analogy

"Imagine a school gate. A traditional guard only stops people on a black-list. Our AI guard watches how people act—if someone tries to climb a wall, they are stopped immediately."

CyberShield AI uses an integrated pipeline that transforms raw network packets into human-readable security advisories. The system processes 41 unique features using Random Forest (ML) and GPT-based summarization (GenAI).

Machine Learning Models

Random Forest

Our primary model. It works like a "Committee of Experts" (Decision Trees) voting on whether a connection is safe or not.

  • 100 Decision Trees
  • Majority Voting Mechanism
  • High Resistance to Overfitting
Deep Learning (DNN)

A complex neural network that finds hidden connections in big data that simpler models might miss.

  • Multi-Layer Perceptron (MLP)
  • 41-Neuron Input Layer
  • Backpropagation Learning

Generative AI Insights

Most AI just says "Attack" or "Normal." Our Generative Insight Engine takes it a step further. It explains why it thinks there is an attack and gives the administrator a 3-step plan to fix it in plain English.

Infrastructure & Security Hardening

A critical part of the college project defense is demonstrating robust system architecture. We have implemented several production-grade security layers:

Auth Guards

Authenticated users are automatically bridged to the dashboard from /UserLogin and /Signup, preventing session loops.

Identity Recovery

Integrated Forgotten Keys modal and /ResetPasswordAction for secure account restoration.

Layering Guard

Z-Index (9999) enforced navigation layering to ensure interactive elements are never masked by dashboard content.

Advanced Multi-User Security

The core system features a production-grade authentication layer to protect administrative functions and user data.

Password Hashing

Uses PBKDF2 with SHA256 to hash passwords. Even if the database is exposed, original passwords remain unreadable.

Ultimate Admin Bypass

A hardcoded admin account providing master access, indicated by a special MASTER badge.

Dataset Features (41 Columns)

The AI doesn't just "see" an attack; it looks at 41 different clues. Here are the core categories:

  • Duration: Length of connection.
  • Protocol_type: TCP, UDP, ICMP.
  • Service: Target service (HTTP, etc.).
  • Flag: Connection status.
  • Src_bytes: Bytes from source.
  • Dst_bytes: Bytes from destination.
  • Land: Same source/destination IP/port.
  • Wrong_fragment: Bad packet fragments.
  • Urgent: Number of urgent packets.

  • Hot: Number of "hot" indicators.
  • Num_failed_logins: Failed attempts.
  • Logged_in: 1 if logged in, else 0.
  • Num_compromised: Compromised states.
  • Root_shell: Root shell obtained.
  • Su_attempted: "su root" command used.
  • Num_root: Number of "root" accesses.
  • Num_file_creations: File ops.
  • Num_shells: Number of shell prompts.
  • Num_access_files: Control files.
  • Num_outbound_cmds: Outbound FTP cmds.
  • Is_host_login: Host login status.
  • Is_guest_login: Guest login status.

  • Count: Connections to same host.
  • Srv_count: Same service connections.
  • Serror_rate: % of "SYN" errors.
  • Srv_serror_rate: Service SYN errors.
  • Rerror_rate: % of "REJ" errors.
  • Srv_rerror_rate: Service REJ errors.
  • Same_srv_rate: % to same service.
  • Diff_srv_rate: % to diff services.
  • Srv_diff_host_rate: % to diff hosts.

  • Dst_host_count: Host count.
  • Dst_host_srv_count: Service count.
  • Dst_host_same_srv_rate: % same srv.
  • Dst_host_diff_srv_rate: % diff srv.
  • Dst_host_same_src_port_rate: % same port.
  • Dst_host_srv_diff_host_rate: % diff host.
  • Dst_host_serror_rate: Host SYN errors.
  • Dst_host_srv_serror_rate: Srv SYN errors.
  • Dst_host_rerror_rate: Host REJ errors.
  • Dst_host_srv_rerror_rate: Srv REJ errors.

Attack Taxonomy

DoS (Denial of Service)

Goal: Crashing the system. (e.g., Neptune, Smurf, Pod)

Probing (Reconnaissance)

Goal: Gathering info. (e.g., Satan, IPsweep, Nmap)

U2R (User to Root)

Goal: Gaining Admin access. (e.g., Buffer Overflow)

R2L (Remote to Local)

Goal: Getting into the system. (e.g., Guess Password)

Viva Preparation

Q: How is user data secured in this application?

A: We use salted PBKDF2 hashing for passwords and a hashed session caching system for local quick access, ensuring the 'admin' account is always secure.

Q: What is the significance of the GenAI layer?

A: It provides Explainable AI (XAI) by translating complex ML predictions into human-readable advice and action plans.

Q: What is the primary objective of CyberShield AI?

A: To proactively detect and classify network intrusions using a multi-layered AI approach, moving from simple detection to actionable generative mitigation plans.

Q: How does the "Liquid Glass" UI enhance security operations?

A: It provides a distraction-free, OLED-optimized interface that reduces cognitive load for security analysts while maintaining high aesthetic standards for reporting.

Q: What are the main challenges faced during data preprocessing?

A: Handling categorical feature encoding (like protocol types) and scaling numerical values (like duration or bytes) to ensure that the AI doesn't get biased by large numbers.

Q: Why use Random Forest?

A: It's an ensemble model that uses many decision trees to vote, making it highly accurate and resistant to memorizing noise (overfitting).

Q: What is Overfitting vs. Underfitting?

A: Overfitting is when a model memorizes the training data too well but fails on new data. Underfitting is when the model is too simple to learn the underlying patterns.

Q: How does Random Forest handle the 41 features of the dataset?

A: It performs feature selection by evaluating "Gini Importance." It prioritizes features like src_bytes and srv_serror_rate that have the highest signal for distinguishing attacks from normal traffic.

Q: What is the role of the Train-Test Split?

A: We split the data (e.g., 80/20) to ensure the model is evaluated on data it has never seen before, which is the only way to prove it can detect future, unknown attacks.

Q: Why use a Random Forest instead of a single Decision Tree?

A: A single tree is prone to "Overfitting." By combining 100+ trees (Ensemble), the Random Forest averages out individual errors, leading to much higher stability and accuracy.

Q: Why call it "Deep" Learning?

A: "Deep" refers to the hidden layers between input and output. More layers allow the network to learn more complex feature representations.

Q: What is the role of the Activation Function?

A: Functions like ReLU or Sigmoid introduce non-linearity, allowing the network to solve problems that aren't just simple straight lines.

Q: What is an Epoch in Deep Learning?

A: One "Epoch" is a full pass of the entire training dataset through the neural network. We use multiple epochs to gradually refine the neuron weights.

Q: How do you prevent Overfitting in Neural Networks?

A: We use techniques like Dropout (randomly turning off neurons during training) to force the network to learn redundant, robust patterns instead of specific data points.

Q: What is the "Loss Function"?

A: It calculates the "Distance" between the AI's prediction and the actual truth. Our goal is to use Optimization (like Adam) to drive this loss to its lowest possible point.

Q: What is "Prompt Engineering"?

A: The art of crafting specific text inputs to guide a Generative AI model toward an accurate or desired output style (e.g., security-focused reports).

Q: What are AI Hallucinations?

A: When a model generates text that sounds confident but is factually incorrect. We mitigate this by using restricted context and specific knowledge bases.

Q: How does GenAI handle Zero-Day (unknown) attacks?

A: While traditional ML detects the anomaly, GenAI analyzes the context of the anomaly to suggest defensive postures even for attacks it has never encountered before.

Q: What is the "Temperature" setting in GenAI?

A: It controls the "Creativity" of the response. For security, we keep it low (e.g., 0.2) to ensure the advice is factual, consistent, and strictly follows security protocols.

Q: Can GenAI replace a human security analyst?

A: No. It acts as a Force Multiplier. It handles the data-heavy analysis and drafting, allowing the human expert to make the final "Execute/Stop" decision faster.

Q: Difference between .py and .ipynb?

A: .py is a linear script. .ipynb is a stateful notebook allowing inline charts, interactive code blocks, and rich documentation.

Q: What is the "Kernel" in Jupyter?

A: The background process that actually executes the code cells. Restarting it clears the environment memory.

Q: Why is Scikit-Learn preferred for this project?

A: It is the gold standard for ML in Python, providing robust, high-performance implementations of Random Forest and standardized tools for feature scaling and evaluation.

Q: How do you manage large datasets in Python without crashing?

A: We use Pandas for efficient memory management and data manipulation, allowing us to process thousands of network logs in seconds.

Q: What is Model Serialization (Joblib)?

A: It is the process of saving the trained "Brain" of the AI to a .joblib file. We specifically use Joblib over standard pickle because it is 3.5x faster at handling large NumPy arrays, ensuring the application loads instantly.
