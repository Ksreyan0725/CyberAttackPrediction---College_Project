Project Overview

This project, "Cyber Attack Prediction: From Traditional ML to Generative AI," is designed to protect digital infrastructure by identifying malicious network traffic patterns before they cause damage.

The Problem

Traditional security systems rely on "signatures" (lists of known bad actors). They fail when a new, unknown attack occurs.

The Solution

Our system uses AI to learn behavior. It can spot a hacker even if it has never seen that specific attack before.

The Security Guard Analogy

"Imagine a school gate. A traditional guard only stops people on a black-list. Our AI guard watches how people act—if someone tries to climb a wall, they are stopped immediately."

System Architecture

// Data Flow Journey
User Upload ➔ Data Pre-processing ➔ ML Prediction ➔ GenAI Explanation

The system is built using Flask (Backend), Bootstrap 5 (Frontend), and Scikit-Learn (AI Engine). It processes 41 unique features for every single network connection attempt.

Machine Learning Models

Random Forest

Our primary model. It works like a "Committee of Experts" (Decision Trees) voting on whether a connection is safe or not.

  • 100 Decision Trees
  • Majority Voting Mechanism
  • High Resistance to Overfitting
Deep Learning (DNN)

A complex neural network that finds hidden connections in big data that simpler models might miss.

  • Multi-Layer Perceptron (MLP)
  • 41-Neuron Input Layer
  • Backpropagation Learning

Generative AI Insights

Most AI just says "Attack" or "Normal." Our Generative Insight Engine takes it a step further. It explains why it thinks there is an attack and gives the administrator a 3-step plan to fix it in plain English.

Dataset Features (41 Columns)

The AI doesn't just "see" an attack; it looks at 41 different clues. Here are the core categories:

  • Duration: Length of the connection (in seconds).
  • Protocol_type: Type of protocol (TCP, UDP, ICMP).
  • Service: Network service on the destination (HTTP, FTP, etc.).
  • Flag: Status of the connection (Normal, Rejected, Error).

  • Src_bytes: Data bytes sent from source to destination.
  • Dst_bytes: Data bytes sent from destination to source.
  • Logged_in: 1 if successfully logged in; 0 otherwise.
  • Num_failed_logins: Number of failed login attempts.

Attack Taxonomy

DoS (Denial of Service)

Goal: Crashing the system. (e.g., Neptune, Smurf, Pod)

Probing (Reconnaissance)

Goal: Gathering info. (e.g., Satan, IPsweep, Nmap)

U2R (User to Root)

Goal: Gaining Admin access. (e.g., Buffer Overflow)

R2L (Remote to Local)

Goal: Getting into the system. (e.g., Guess Password)

Viva Preparation

Q: Why use Random Forest?

A: It's an ensemble model that uses many decision trees to vote, making it highly accurate and resistant to memorizing noise (overfitting).

Q: What is the significance of the GenAI layer?

A: It provides Explainable AI (XAI) by translating complex ML predictions into human-readable advice and action plans.
